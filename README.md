# Agent Skills

一个用于 [Zed](https://zed.dev) 编码智能体（Agent）的技能（Skill）集合。

技能是模块化的能力单元，为特定任务提供专门的指令。当用户的请求匹配某个技能的描述时，智能体会自动加载该技能的完整指令并遵循执行，从而在特定场景下表现得更规范、更可靠。

## 目录结构

```
.
├── LICENSE
├── README.md
└── skills/
    └── <skill-name>/
        └── SKILL.md
```

每个技能位于 `skills/` 下的独立目录中，核心文件为 `SKILL.md`。技能目录还可以包含其所需的其他资源文件（如脚本、模板、许可证等），技能内引用的相对路径均以该技能目录为根进行解析。

## SKILL.md 格式

`SKILL.md` 由 YAML frontmatter 和 Markdown 正文两部分组成：

```markdown
---
name: skill-name
description: "对该技能用途的清晰描述，用于让智能体判断何时应加载此技能。"
---

# 技能正文

在这里编写该技能的详细指令。
```

- `name`：技能的唯一标识，通常与所在目录名一致。
- `description`：技能的用途说明。这是智能体决定是否触发该技能的关键依据，应尽量准确地描述适用场景。
- 正文：技能被加载后，智能体需要遵循的具体指令。

## 使用方式

将本仓库的技能放置在 Zed 智能体可识别的技能目录下即可。当对话中的请求与某个技能的 `description` 匹配时，智能体会自动检索并应用相应技能。

## 新增技能

1. 在 `skills/` 下创建一个新目录，目录名即技能名。
2. 在该目录中创建 `SKILL.md`，填写 frontmatter 与正文指令。
3. 如有需要，添加技能所需的辅助文件。
4. 确保 `description` 准确描述适用场景，以便智能体在合适的时机触发。
